# 架构设计学习指南

## 模块概览

本模块涵盖架构设计三大核心方向：**设计模式**、**分布式核心**、**DDD 领域驱动设计**。
统一使用「电商支付系统」作为业务场景，所有设计模式都在真实上下文中演示。

---

## 一、设计模式（6 种高频模式）

### 1.1 策略模式 — 支付方式选择

| 文件 | 说明 |
|------|------|
| `PayStrategy.java` | 策略接口：定义 pay()、getChannel()、calculateFee() |
| `AlipayStrategy.java` | 支付宝实现：0.6% 手续费，5 万限额 |
| `WxPayStrategy.java` | 微信实现：0.6% 手续费，5000 单笔限额 |
| `BankPayStrategy.java` | 银行卡实现：1.0% 手续费 |
| `PayContext.java` | 上下文：Map 注入所有策略，按 channel 选择 |

**核心知识点：**
- Spring 中用 `List<T>` 注入 + `Collectors.toMap()` 替代 if-else
- 新增支付方式只需加 `@Component` 类，Context 零修改（开闭原则）

**测试接口：**
```
GET /api/architecture/strategy/pay?channel=ALIPAY&amount=100&orderId=ORD-001
GET /api/architecture/strategy/channels
```

---

### 1.2 工厂模式 — 订单创建

| 文件 | 说明 |
|------|------|
| `Order.java` | sealed 接口 + 3 种订单实现（Normal/FlashSale/Group） |
| `CreateOrderRequest.java` | 创建请求 Record |
| `OrderCreatorImpl.java` | 3 个创建器（内部类），各自校验逻辑 |
| `OrderFactory.java` | 工厂：Map 注入，按 type 选择创建器 |

**核心知识点：**
- `sealed interface` 限制继承层级（Java 17）
- `record` 作为不可变 DTO
- 工厂模式 vs 策略模式：工厂关注"创建谁"，策略关注"怎么做"

**测试接口：**
```
POST /api/architecture/factory/order
Body: {"type":"NORMAL","userId":"U001","productId":"P001","quantity":2,"unitPrice":99.9}
```

---

### 1.3 观察者模式 — 事件驱动

| 文件 | 说明 |
|------|------|
| `OrderEvent.java` | sealed 事件接口 + Created/Paid/Cancelled |
| `OrderEventListener.java` | 3 个监听器：短信、库存、日志 |
| `OrderEventPublisher.java` | 事件发布者 |

**核心知识点：**
- Spring Event：`@EventListener` + `ApplicationEventPublisher`
- `@Async` 异步监听（需 `@EnableAsync`）
- 发布者和订阅者完全解耦

**测试接口：**
```
GET /api/architecture/observer/order-created?orderId=ORD-001&userId=U001&productId=P001&quantity=2&amount=199.8
GET /api/architecture/observer/order-paid?orderId=ORD-001&userId=U001&amount=199.8
```

---

### 1.4 模板方法 — 数据导出

| 文件 | 说明 |
|------|------|
| `ExportService.java` | 抽象模板：定义 export() 流程骨架 |
| `ExportServiceImpl.java` | 3 个实现：Excel/CSV/JSON |

**核心知识点：**
- `final` 模板方法防止子类修改流程
- 抽象方法（必须重写）vs 钩�方法（可选重写，如 `needUpload()`）
- 模板方法 vs 策略模式：模板方法是"流程固定、步骤可变"，策略是"算法整体替换"

---

### 1.5 责任链模式 — 订单校验

| 文件 | 说明 |
|------|------|
| `OrderFilter.java` | 过滤器接口：filter() + getOrder() |
| `OrderFilterImpl.java` | 4 个过滤器：参数→库存→风控→优惠券 |
| `OrderFilterChain.java` | 过滤链：自动注入、按优先级排序、短路执行 |

**核心知识点：**
- Spring 自动注入 `List<OrderFilter>`，按 `getOrder()` 排序
- 短路机制：遇到失败立即中断
- 新增校验器只需加 `@Component`，链零修改

**测试接口：**
```
GET /api/architecture/chain/validate?userId=U001&productId=P001&quantity=2&amount=199.8&userIp=192.168.1.1
GET /api/architecture/chain/validate?userId=U001&productId=P002&quantity=1&amount=99.9&userIp=192.168.1.100
（第二个会触发风控拦截）
```

---

### 1.6 装饰器模式 — 服务增强

| 文件 | 说明 |
|------|------|
| `CacheableUserService.java` | 接口 + 真实实现 + CacheDecorator + LogDecorator |

**核心知识点：**
- 装饰器和被装饰者实现同一接口
- 可以多层嵌套：`new LogDecorator(new CacheDecorator(realService))`
- 装饰器 vs 代理模式：装饰器关注"增强功能"，代理关注"控制访问"
- Spring AOP 的底层思想

## 二、分布式核心

### 2.1 分布式锁 — Redisson

| 文件 | 说明 |
|------|------|
| `DistributedLockDemo.java` | 5 种锁用法：基本锁、tryLock、看门狗、公平锁、读写锁 |

**核心知识点：**
- Redisson 看门狗（Watchdog）：每 10 秒自动续期
- `tryLock(waitTime, leaseTime, unit)`：非阻塞尝试
- 公平锁：按请求顺序获取，防止饥饿
- 读写锁：读锁共享、写锁独占

---

### 2.2 分布式 ID — 雪花算法

| 文件 | 说明 |
|------|------|
| `IdGenerator.java` | 3 种实现：UUID、Snowflake、Increment |

**核心知识点：**
- 雪花算法 ID 结构：1 符号 + 41 时间戳 + 10 机器 + 12 序列
- 时钟回拨处理
- 每毫秒可生成 4096 个 ID

**测试接口：**
```
GET /api/architecture/distributed-id?type=snowflake&count=5
GET /api/architecture/distributed-id?type=uuid&count=5
```

---

### 2.3 分布式事务 — TCC/Saga/本地消息表

| 文件 | 说明 |
|------|------|
| `DistributedTransactionDemo.java` | 3 种方案：TCC、Saga、本地消息表 |

**核心知识点：**
- TCC：Try 预留 → Confirm 确认 → Cancel 回滚
- Saga：正向操作 + 补偿操作（适合长流程）
- 本地消息表：业务+消息同一事务，轮询发送

## 三、DDD 领域驱动设计

### 3.1 核心概念

| 文件 | 说明 |
|------|------|
| `OrderAggregate.java` | 聚合根 + 值对象(Money) + 实体(OrderItem) + 领域事件 |
| `OrderDomainService.java` | 领域服务 + 应用服务 |

**核心知识点：**
- 值对象（Money）：不可变、无 ID、通过值判断相等
- 实体（OrderItem）：有 ID、有业务逻辑
- 聚合根（OrderAggregate）：维护业务不变量、控制内部访问
- 领域事件：状态变更时发布，解耦聚合
- 贫血模型 vs 充血模型

**测试接口：**
```
GET /api/architecture/ddd/create-order?userId=U001&productId=P001&productName=iPhone&quantity=1&price=7999&userLevel=VIP
```

## 四、设计模式对比总结

| 模式 | 解决的问题 | Spring 中的实现方式 |
|------|-----------|-------------------|
| 策略 | 算法替换 | `List<T>` 注入 + Map |
| 工厂 | 对象创建 | `List<T>` 注入 + Map |
| 观察者 | 事件驱动 | Spring Event |
| 模板方法 | 流程固定、步骤可变 | 抽象类 + 继承 |
| 责任链 | 多级处理 | `List<T>` 注入 + 排序 |
| 装饰器 | 动态增强 | 接口 + 组合 |

## 五、学习建议

1. **先跑通接口**：启动项目，调用 `/api/architecture/*` 接口，观察日志
2. **读代码注释**：每个文件都有详细的注释，包括类图、适用场景、对比分析
3. **对比学习**：策略 vs 工厂、模板方法 vs 策略、装饰器 vs 代理
4. **动手改造**：尝试新增一种支付方式、新增一个过滤器，体会开闭原则
5. **联系实际**：想想自己项目中哪些地方可以用这些模式
